6 · CLI: onboarding sequence and command copy

6.1 First-visit onboarding (about 12 seconds, respect prefers-reduced-motion)

  1. Show prompt: akash@site:~$ cursor.
  2. Type help then execute; show output from 6.3.
  3. Backspace to clear (skip if reduced motion).
  4. Type open work --filter=ai then execute; show opening /work#ai plus link.
  5. CTAs: try the tour (switch to chat + highlight chips) · dismiss · never show again (sets localStorage).
CLI section title / sub
cli mode / typed commands. no small talk. if you already live in a terminal, this is the fast path.

6.2 Errors

  • unknown command. type help.
  • no page at that path. try open work.

6.3 help output (exact)

commands you will actually use:

  help              show this
  open <path>       jump to a page. examples: open work, open library
  list <thing>      list items. examples: list builds, list pdfs
  find "<query>"      search titles across the site

flags:

  --filter=<tag>    narrow work or library. examples: open work --filter=ai

tip: same intents work in chat. cli is just explicit routing.

6.4 Command dictionary (exact)

open <path> [--filter=<tag>]
  paths: /, /work, /writing, /builds, /library, /about, /contact
  example: open work --filter=ai

list <target> [--domain=<tag>] [--type=<type>]
  targets: builds | pdfs | threads
  example: list pdfs --domain=ai

find "<query>"
  example: find "routing"

clear
  clears transcript

help
  shows commands
